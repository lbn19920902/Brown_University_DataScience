1. Human Resources Analytics Solution¶

Define problem

Build model to predict whether the employee left or not by using given data

Metric

AUC(Area under the curve)

Data Description

Publish with us on Medium at https://medium.com/towards-data-science

Why are our best and most experienced employees leaving prematurely? Have fun with this database and try to predict which valuable employees will leave next. Fields in the dataset include:

Employee satisfaction level Last evaluation Number of projects Average monthly hours Time spent at the company Whether they have had a work accident Whether they have had a promotion in the last 5 years Department Salary Whether the employee has left *This dataset is simulated

Column Description

satisfaction_level Level of satisfaction (0-1)

last_evaluation Evaluation of employee performance (0-1)

number_project Number of projects completed while at work

average_montly_hours Average monthly hours at workplace

time_spend_company Number of years spent in the company

Work_accident Whether the employee had a workplace accident

left Whether the employee left the workplace or not (1 or 0) Factor

promotion_last_5years Whether the employee was promoted in the last five years

sales Department in which they work for

salary Relative level of salary (low med high)

def onehot(data,column_name):
    dummies=pd.get_dummies(data[column_name],prefix="_"+column_name)
    data=data.join(dummies)
    data=data.drop([column_name],axis=1)
    return data

5.Summary¶

1. Data: The data set we have is in a good quality. For example, there is no missing data, no skewness and already been standardized. The scale of data is small, it only use 1 MB memory.

2. Model: Because our data is simulated, in this case, the tree algorithm and clustering algorithm showed their strenghts.

3. To Improve: I did not create any feature, because there are not many meaningful feature i can create. However, in feature average_monthly_hours, last_evaulation and satisfation_level there are some ranges have absolute low probability for left=0. We can bin these if necessary. Tree based models and clustering models work really good, they reach high AUC score in this case. We can create second level to reweight each model, however i do not think we need to do that, as XGBoost and SVM reach AUC=0.96 and KNN's AUC=0.94

4. Overall: It is a really interesting data set to work on.
